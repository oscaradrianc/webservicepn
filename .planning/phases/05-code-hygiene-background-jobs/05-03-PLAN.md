- Implement `ProcessWithRetryAsync` with exactly 3 attempts, 2000ms delay:
  ```csharp
  private async Task ProcessWithRetryAsync(EmailMessage message, CancellationToken ct)
  {
      const int maxAttempts = 3;
      const int delayMs = 2000;
      for (int attempt = 1; attempt <= maxAttempts; attempt++)
      {
          try
          {
              SendMailInternal(message);
              return;
          }
          catch (Exception ex) when (attempt < maxAttempts)
          {
              _logger.LogWarning(ex,
                  "SendMail attempt {Attempt}/{Max} failed. Recipients: {Recipients}, Subject: {Subject}",
                  attempt, maxAttempts, message.Recipients, message.Subject);
              await Task.Delay(delayMs, ct);
          }
          catch (Exception ex)
          {
              _logger.LogError(ex,
                  "SendMail permanently failed after {Max} attempts. Recipients: {Recipients}, Subject: {Subject}",
                  maxAttempts, message.Recipients, message.Subject);
          }
      }
  }
  ```
- Implement `GetSmtpConfig()` using IMemoryCache with 5-minute TTL:

  private record SmtpConfig(string Server, int Port, string Username,

      return _cache.GetOrCreate("smtp_config", entry =>
      {
          entry.AbsoluteExpirationRelativeToNow = TimeSpan.FromMinutes(5);

              Ssl:      Convert.ToBoolean(cx.POGECONSTANTEs.Where(x => x.CONSREFERENCIA == "ssl_mail").Select(x => x.CONSVALOR).First())

  ```csharp
  private void SendMailInternal(EmailMessage message)
  {
      var cfg = GetSmtpConfig();
      using var mail = new MailMessage();
      mail.From = new MailAddress(cfg.Sender);
      if (message.Bcc)
          message.Recipients.ForEach(r => mail.Bcc.Add(r));
      else
          message.Recipients.ForEach(r => mail.To.Add(r));
      mail.Subject = message.Subject;
      mail.Body = message.Body;
      mail.IsBodyHtml = true;
      using var smtp = new SmtpClient(cfg.Server);
      smtp.Port = cfg.Port;
      smtp.Credentials = new System.Net.NetworkCredential(cfg.Username, cfg.Password);
      smtp.EnableSsl = cfg.Ssl;
      smtp.Send(mail);
  }
  ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| BusinessService → IEmailQueue.Queue() | EmailMessage data crosses from request thread to background queue; no external input surfaces added |
| EmailQueueService → SmtpClient | Email dispatch uses config from Oracle via IMemoryCache; no user-supplied SMTP settings |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-03-01 | Information Disclosure | EmailQueueService logs — Recipients and Subject logged on failure | accept | Log entries contain email addresses (PII). Acceptable: logs are server-side only, accessed by operators. No external exposure. |
| T-05-03-02 | Denial of Service | Unbounded Channel<EmailMessage> memory growth | accept | Email volume is low (10 call sites, infrequent triggers). Unbounded channel will not grow significantly. If volume increases unexpectedly, bounded channel with drop logging can be introduced in v2. |
| T-05-03-03 | Tampering | SMTP config IMemoryCache — in-process cache, not persisted | accept | Cache is populated from Oracle DB values. Cannot be externally poisoned without Oracle access. TTL is 5 minutes — config changes propagate within 5 minutes. |
| T-05-03-04 | Information Disclosure | SMTP password in SmtpConfig record — in-process memory | accept | Password is in process memory during operation, same as before. No new exposure — previously was in direct Oracle query result sets. IMemoryCache reduces the number of DB round-trips containing the password. |
</threat_model>
