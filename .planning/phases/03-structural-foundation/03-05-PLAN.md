**Step 2 — Update Login.cs constructor (per D-05):**


To:
```csharp
private readonly IUtilidades _utilidades;

    _configuration = configuration;
    _factory = factory;
}
```

**Step 3 — Update all three method signatures in Login.cs:**

Remove `IConfiguration configuration` parameter from Authenticate, ChangePassword, ResetPassword. They now use `_configuration` from the field.

Example — Authenticate signature changes from:
```csharp
public Response<Usuario> Authenticate(LoginRequest login, IConfiguration configuration)
```
To:
```csharp
public Response<Usuario> Authenticate(LoginRequest login)
```

Apply same change to ChangePassword and ResetPassword.

Inside method bodies: all references to the `configuration` parameter become `_configuration` (no behavior change — same object).

**Step 4 — Fix DataContext instantiation in Login.cs (per D-07 and DAT-09):**

Authenticate currently uses: `PORTALNEGOCIODataContext dc = new PORTALNEGOCIODataContext();` without using.
ChangePassword currently uses: `PORTALNEGOCIODataContext dc = new PORTALNEGOCIODataContext();` without using.
ResetPassword currently uses: `PORTALNEGOCIODataContext cx = new PORTALNEGOCIODataContext();` without using.

Replace all three with `using (var cx = _factory.Create())` / `using (var dc = _factory.Create())` blocks, wrapping the code that uses the context.

For ResetPassword specifically — wrap the entire method body that uses cx in a using block:
```csharp
public ResponseStatus ResetPassword(ResetPassRequest req)
{
    ResponseStatus resp = new ResponseStatus();
    using (var cx = _factory.Create())
    {
        var tblUsuario = (from u in cx.POGEUSUARIOs ...).FirstOrDefault();
        // ... all existing logic using cx ...
        cx.SubmitChanges();
    }
    return resp;
}
```
