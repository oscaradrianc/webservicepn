<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| appsettings.json → Serilog log levels | Los niveles de filtrado vienen de config — un cambio erróneo puede activar logging de Debug que exponga info sensible |
| outputTemplate → log file | El template define qué campos de cada log event van al archivo — incluye {Exception} que puede contener stack traces |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02-01 | Information Disclosure | outputTemplate con {Exception} en File sink | accept | Stack traces en logs de servidor son aceptables — el archivo de log es local al servidor. No enviar al cliente. ASVS L1 requiere que los errores no lleguen al usuario, no que no estén en los logs internos. |
| T-02-02-02 | Information Disclosure | MinimumLevel.Default = Information | accept | Nivel Information puede loguear datos de negocio en controllers/services. Aceptable: el log file es server-side. Passwords y tokens no deben llegar a nivel Information por diseño del código. |
| T-02-02-03 | Tampering | Namespace Override "Devart": "Warning" | accept | Subir el nivel de Devart evita que se loguen queries SQL con parámetros. Esto REDUCE exposición accidental de datos de BD. Disposición: correcto por diseño. |
</threat_model>
